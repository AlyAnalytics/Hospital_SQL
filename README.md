# Healthcare Clinic Operations Analysis 🏥

## 📌 Project Overview
This repository contains my final SQL data analysis project for the "Data Analysis Skills for the Freelance Professional" program at the National Telecommunication Institute (NTI). 

The objective of this project was to design, structure, and analyze a relational healthcare database. The dataset encompasses clinic operations, including patient demographics, doctor scheduling, department performance, and medical record tracking. 

## 🛠️ Tools & Techniques
* **Database Management System:** Microsoft SQL Server
* **SQL Techniques Mastered & Applied:** 
  * Data Definition Language (DDL) for establishing Primary and Foreign Key constraints
  * Advanced Joins & Relational Mapping
  * Common Table Expressions (CTEs) for query optimization
  * Window Functions (`DENSE_RANK()`, `ROW_NUMBER()`)
  * Data Cleaning and Transformation (handling duplicate records and standardizing text)
  * Aggregations, Grouping, and Date/Time logic

## 📂 Repository Structure
* `Healthcare_Analysis_Final.sql`: The complete SQL script containing schema setup, data cleaning procedures, and 15 highly targeted analytical queries.

## 💡 Key Business Questions Answered
Through complex SQL querying, I extracted actionable insights to solve real-world clinic management problems:
1. **Financial Performance:** Calculated the total revenue generated by each doctor based on completed appointments.
2. **Operational Efficiency:** Identified the busiest departments by appointment count to help optimize staffing.
3. **Patient Demographics:** Developed automated age categorization (Pediatric, Adult, Senior) and executed targeted queries for marketing campaigns.
4. **Medical Tracking:** Optimized queries using CTEs to track specific medication usage (e.g., Metformin) and identify patients requiring follow-up.

## 📊 Code Highlight: Advanced Window Functions
To find the most experienced specialist within every single department, I utilized a CTE combined with a `DENSE_RANK()` window function:

```sql
;WITH RankedDoctors AS (
    SELECT 
        dep.department_name,
        doc.full_name AS doctor_name,
        doc.years_experience,
        DENSE_RANK() OVER (
            PARTITION BY dep.department_name 
            ORDER BY doc.years_experience DESC
        ) AS experience_rank
    FROM doctors doc
    JOIN departments dep ON doc.department_id = dep.department_id
)
SELECT 
        department_name, 
        doctor_name, 
        years_experience
FROM RankedDoctors
WHERE experience_rank = 1;
